## Integration Check Results

**Cross-phase wiring: 18/18 connected** — Every phase's key exports are consumed by downstream phases. No orphaned exports, no missing connections.

**API route coverage: 5/5 consumed:**
- `GET /health` → setup.ts:verifyChain()
- `GET /tabs` → search-tabs.tsx:fetchAllTabs() (with pagination via ?offset=)
- `POST /switch` → search-tabs.tsx:switchTab()
- `POST /close` → search-tabs.tsx:closeTab()
- `GET /favicon?url=` → search-tabs.tsx favicon fetch

**E2E Flows: 5/5 complete:**

1. **Tab list load** — Raycast invokes → getPort() reads ~/.raycast-firefox/port → GET /tabs → server.js:handleGetTabs → bridge.js:sendRequest("list-tabs") → native protocol → background.js:handleListTabs → browser.tabs.query({}) → response back through chain → List.Item render
2. **Switch tab** — Action "Switch to Tab" → closeMainWindow → fetch POST /switch {tabId, windowId} → sendRequest("switch-tab") → browser.tabs.update + browser.windows.update → osascript Firefox foreground
3. **Close tab** — Action "Close Tab" (Ctrl+X) → mutate with optimisticUpdate (filter out tab) → fetch POST /close {tabId} → sendRequest("close-tab") → browser.tabs.remove → Toast "Tab closed"
4. **Error recovery** — Three classified paths:
   - FirefoxNotRunning: port-file-missing + !isFirefoxRunning → "Launch Firefox" action → execFile("open", ["-a", "Firefox"])
   - ExtensionNotInstalled: bridge "not connected" or timeout → "Install WebExtension" → opens AMO page
   - HostNotRunning: port-file-missing + isFirefoxRunning → "Set Up Native Host" → launchCommand("setup-bridge")
5. **Setup command** — setup-bridge.tsx → isFirefoxInstalled → resolveNativeHostPath (reads project-root.txt asset) → generateManifest → writeManifest → validateManifest → verifyChain → HUD/toast feedback

**Broken flows:** 0

### Integration Nuances (Non-Blocking)

1. **ERRH-02 string coupling:** classifyError matches `msg.includes("not connected")` against bridge.js error text. Intentional and documented in errors.ts comments, but fragile to message rewording.
2. **windowId in POST /switch:** Forwarded through full chain but background.js derives windowId from tab update result instead of using the parameter. Correct behavior, unused payload.
3. **project-root.txt:** Contains current dev machine path, regenerated at build time via prebuild script. Committed value stale for other developers building from checkout.
