**2. ClassifiedError interface:**
```typescript
export interface ClassifiedError {
  mode: FailureMode;
  title: string;
  description: string;
}
```

**3. isFirefoxRunning() function:**
- Uses `execFileSync("pgrep", ["-xi", "firefox"], { stdio: "ignore" })` from `child_process`
- Returns `true` if pgrep exits 0 (process found), `false` on any error (exit code 1 = not found)
- IMPORTANT: Only call this inside async functions (off the render thread), never in component body

**4. classifyError(error: unknown): ClassifiedError function:**
Decision tree (per RESEARCH.md):
- If `error instanceof TypeError` and `error.message === "fetch failed"`:
  - Check `(error as any).cause?.code === "ECONNREFUSED"` (MUST check `.cause.code`, NOT `.code`)
  - If ECONNREFUSED and `!isFirefoxRunning()`: return FirefoxNotRunning with title "Firefox Isn't Running", description "Launch Firefox to see your tabs here"
  - If ECONNREFUSED and Firefox IS running: return HostNotRunning with title "Native Host Not Connected", description "The Raycast Firefox helper needs to be set up. Make sure the companion extension is installed and the native host is registered."
- If `error instanceof Error` and message includes "not connected" or "timeout" (case-insensitive): return ExtensionNotInstalled with title "WebExtension Not Connected", description "The Firefox companion extension isn't responding. Make sure it's installed and Firefox is running."
- Otherwise: return Unknown with title "Can't Connect to Firefox", description "Something unexpected went wrong. Try restarting Firefox."

Note: Per RESEARCH.md key insight, when Firefox is running + ECONNREFUSED, both "extension not installed" and "host not registered" are indistinguishable. Use a combined message for HostNotRunning covering both possibilities.

**5. fetchWithRetry<T>(fn, maxRetries?, baseDelayMs?): Promise<T>:**
- Default: 3 retries, 1000ms base delay
- Exponential backoff: delay = baseDelayMs * 2^attempt (1s, 2s, 4s)
- Total max wait before error: ~7 seconds
- Catches each attempt, waits, retries. After all retries exhausted, throws the last error.

**6. showActionError(error: unknown, actionName: string): Promise<void>:**
- Calls `classifyError(error)` to get the classified error
- Shows a Raycast toast with `Toast.Style.Failure`
- Title: `Couldn't ${actionName}` (e.g., "Couldn't switch tab", "Couldn't close tab")
- Message: `classified.description`
- If mode is FirefoxNotRunning: add `primaryAction` with title "Launch Firefox" that runs `execFile("open", ["-a", "Firefox"])` then hides the toast
- Import `showToast`, `Toast` from `@raycast/api` and `execFile` from `child_process`

Tone per user decision: friendly and helpful, not technical error codes.
