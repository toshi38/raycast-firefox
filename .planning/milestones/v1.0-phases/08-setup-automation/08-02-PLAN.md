In `raycast-extension/src/search-tabs.tsx`, update the `HostNotRunning` action in the `List.EmptyView` actions section.

**Current code** (lines ~515-527):
```tsx
{classifiedError.mode === FailureMode.HostNotRunning && (
  <Action
    title="Set Up Native Host"
    icon={Icon.Terminal}
    onAction={async () => {
      await showToast({
        style: Toast.Style.Failure,
        title: "Setup Not Available Yet",
        message: "Install the companion extension in Firefox and register the native host manually",
      });
    }}
  />
)}
```

**Replace with:**
```tsx
{classifiedError.mode === FailureMode.HostNotRunning && (
  <Action
    title="Set Up Native Host"
    icon={Icon.Terminal}
    onAction={async () => {
      await launchCommand({ name: "setup-bridge", type: LaunchType.UserInitiated });
    }}
  />
)}
```

**Also add imports** at the top of the file:
- Add `launchCommand, LaunchType` to the `@raycast/api` import

Remove the `showToast` call that was in the old HostNotRunning action (showToast is still imported for other uses in the file, like closeTab — keep the import).

This is a minimal change — only the `onAction` handler body changes, and two imports are added.
