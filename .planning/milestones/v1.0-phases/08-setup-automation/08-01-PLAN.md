<task type="auto">
  <name>Task 1: Create setup utility module</name>
  <files>raycast-extension/src/lib/setup.ts</files>
  <action>
Create `raycast-extension/src/lib/setup.ts` with these functions:

**`isFirefoxInstalled(): boolean`**
- Check if `/Applications/Firefox.app` exists using `existsSync`
- Simple path check is sufficient (macOS only, per CONTEXT.md decision)

**`resolveNativeHostPath(): string`**
- Resolve the absolute path to `native-host/run.sh` from the Raycast extension's runtime location
- Use `__dirname` to get the current module's directory (will be inside `raycast-extension/` at runtime)
- Walk up from the compiled output directory to find the project root
- Strategy: from `__dirname`, walk up directories until we find a directory containing `native-host/run.sh`
- Cap the walk at 10 levels to avoid infinite traversal
- Throw descriptive error if `run.sh` not found: "Could not locate native-host/run.sh. Ensure the extension is installed correctly."

**`generateManifest(runShPath: string): string`**
- Return JSON string for the native messaging host manifest:
  ```json
  {
    "name": "raycast_firefox",
    "description": "Raycast Firefox tab management bridge",
    "path": "<runShPath>",
    "type": "stdio",
    "allowed_extensions": ["raycast-firefox@lau.engineering"]
  }
  ```
- Use `JSON.stringify` with 2-space indent for readability

**`writeManifest(manifestJson: string): string`**
- Target path: `~/Library/Application Support/Mozilla/NativeMessagingHosts/raycast_firefox.json`
- Use `os.homedir()` to resolve `~`
- Create the `NativeMessagingHosts` directory if it doesn't exist (`mkdirSync` with `recursive: true`)
- Write the manifest file
- Return the absolute path where it was written

**`validateManifest(manifestPath: string, expectedRunShPath: string): { valid: boolean; error?: string }`**
- Read the manifest file back
- Parse as JSON
- Verify `name` is "raycast_firefox"
- Verify `path` matches `expectedRunShPath`
- Verify `path` points to an existing, executable file (check with `accessSync` using `fs.constants.X_OK`)
- Return `{ valid: true }` or `{ valid: false, error: "..." }`

**`verifyChain(port: number): Promise<{ reachable: boolean; tabsOk: boolean }>`**
- Try `fetch(`http://127.0.0.1:${port}/health`)` with a 3-second timeout (`AbortSignal.timeout(3000)`)
- If health responds OK, try `/tabs` endpoint
- Return `{ reachable: true, tabsOk: true|false }` or `{ reachable: false, tabsOk: false }`
- Catch all errors gracefully (connection refused, timeout, etc.)

**`getPort(): number`**
- Read port from `~/.raycast-firefox/port` file (same logic as in search-tabs.tsx)
- Fall back to 26394

Export all functions. Use Node.js built-ins only (fs, path, os) — no new dependencies needed.

**Step B: Create `raycast-extension/src/setup-bridge.tsx`**

This is a `no-view` command — the default export is an async function that runs and exits (no React component).

```typescript
import { showToast, Toast, showHUD } from "@raycast/api";
```

Import all needed functions from `./lib/setup`.

**Flow:**

1. **Pre-flight: Firefox installed?**
   - Call `isFirefoxInstalled()`
   - If false: show failure toast "Firefox Not Detected" with message "Install Firefox first, then re-run this command." and return early.

2. **Resolve native host path**
   - Call `resolveNativeHostPath()`
   - If throws: show failure toast "Setup Failed" with the error message and return early.

3. **Generate and write manifest**
   - Call `generateManifest(runShPath)` then `writeManifest(manifestJson)`
   - If throws (permission error, etc.): show failure toast "Couldn't Write Manifest" with message "Check permissions on ~/Library/Application Support/Mozilla/" and return early.

4. **Validate manifest**
   - Call `validateManifest(manifestPath, runShPath)`
   - If not valid: show failure toast "Manifest Validation Failed" with the error from validation and return early.

5. **Chain verification (best-effort)**
   - Call `getPort()` then `verifyChain(port)`
   - If `reachable && tabsOk`: show HUD "Firefox integration ready!" (green checkmark via showHUD — per CONTEXT.md decision, Raycast closes)
   - If `reachable && !tabsOk`: show **failure** toast with title "Setup Incomplete" and message "Host not responding. Ensure Firefox is running and the companion extension is installed." (per CONTEXT.md: host not responding is treated as failure with guidance)
   - If `!reachable`: show success toast "Manifest Installed" with message "Start Firefox with the companion extension to complete setup." (per CONTEXT.md: Firefox not running is not a failure — manifest is installed correctly)

**Error handling:** Wrap the entire flow in try/catch. On unexpected error, show failure toast "Setup Failed" with error.message.

**Style:** Follow existing patterns from search-tabs.tsx. Use Toast.Style.Failure for errors, Toast.Style.Success for partial success, showHUD for complete success.
