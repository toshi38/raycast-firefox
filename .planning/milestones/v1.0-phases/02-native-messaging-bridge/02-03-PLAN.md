<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: End-to-end round-trip verification</name>
  <what-built>
    Complete native messaging bridge: HTTP server on localhost, native messaging protocol over stdin/stdout, request-response correlation, PID/port file management, and install script. The WebExtension (Phase 1) connects to the native host and handles list-tabs, switch-tab, and close-tab commands.
  </what-built>
  <how-to-verify>
    **Setup (one-time):**
    1. Ensure native messaging manifest is installed: `cat ~/Library/Application\ Support/Mozilla/NativeMessagingHosts/raycast_firefox.json`
       - If missing, run: `bash native-host/install.sh`
    2. Open Firefox
    3. Load the extension via about:debugging -> This Firefox -> Load Temporary Add-on -> select `extension/manifest.json`
    4. Open 2-3 tabs in Firefox with recognizable URLs (e.g., github.com, google.com)

    **Test 1: Health check**
    ```bash
    curl -s http://127.0.0.1:26394/health | python3 -m json.tool
    ```
    Expected: JSON with `ok: true`, `firefoxConnected: true`, `port: 26394`, `version: "1.0.0"`

    **Test 2: List tabs**
    ```bash
    curl -s http://127.0.0.1:26394/tabs | python3 -m json.tool
    ```
    Expected: JSON with `ok: true`, `data.tabs` array containing your open Firefox tabs with titles and URLs

    **Test 3: Switch tab**
    Pick a tabId from the /tabs response and run:
    ```bash
    curl -s -X POST http://127.0.0.1:26394/switch -H 'Content-Type: application/json' -d '{"tabId": TAB_ID_HERE}' | python3 -m json.tool
    ```
    Expected: Firefox switches to that tab and its window comes to front. Response has `ok: true`.

    **Test 4: Error handling**
    ```bash
    curl -s -X POST http://127.0.0.1:26394/switch -H 'Content-Type: application/json' -d '{}' | python3 -m json.tool
    ```
    Expected: 400 error about missing tabId

    **Test 5: Check debug page (optional)**
    In Firefox, navigate to the extension's debug page (from about:debugging, click "Inspect" on the extension, find the debug.html URL). Should show connection status as connected and recent messages in the log.
  </how-to-verify>
