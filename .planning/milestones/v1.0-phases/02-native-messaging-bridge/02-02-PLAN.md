  <name>Task 1: Create lifecycle management and request-response bridge</name>
  <files>

  <action>
    Create `native-host/src/lifecycle.js`:
    - Import logger from ./logger.js, path, os, fs
    - Constants: CONFIG_DIR = path.join(os.homedir(), '.raycast-firefox'), PID_FILE = CONFIG_DIR/host.pid, PORT_FILE = CONFIG_DIR/port
    - `killOldProcess()`:
      - Read PID file, parse as integer
      - If file doesn't exist (ENOENT), return (first run)
      - If PID is NaN, delete stale file and return
      - Use process.kill(oldPid, 0) to check if process exists
      - If exists: log "Killing old host process" with pid, send SIGTERM
      - Wait 1 second (synchronous sleep via Atomics.wait or setTimeout with await), then check again
      - If still alive after 1s: send SIGKILL, log warning
      - If ESRCH: log "Stale PID file, old process already gone"
    - `writePidFile()`:
      - fs.mkdirSync(CONFIG_DIR, { recursive: true })
      - fs.writeFileSync(PID_FILE, String(process.pid))
      - Log "PID file written" with pid
    - `cleanupPidFile()`:
      - try fs.unlinkSync(PID_FILE); catch and ignore
    - `writePortFile(port)`:
      - Write to a temp file in CONFIG_DIR first, then fs.renameSync to PORT_FILE (atomic on POSIX)
      - Log "Port file written" with port
    - `cleanupPortFile()`:
      - try fs.unlinkSync(PORT_FILE); catch and ignore
    - `getConfigDir()`: returns CONFIG_DIR
    - Export all functions

    Create `native-host/src/bridge.js`:
    - Import crypto, logger
    - `pendingRequests` Map: id -> { resolve, reject, timer }
    - Module-level variable: `sendNativeMessageFn` = null (set by host.js)
    - Module-level variable: `nativeConnected` = false (set by host.js)
    - `setSendNativeMessage(fn)`: sets the sendNativeMessageFn
    - `setNativeConnected(connected)`: sets nativeConnected
    - `isNativeConnected()`: returns nativeConnected
    - `sendRequest(command, params = {})`:
      - If !nativeConnected, reject with Error("Firefox is not connected. Is the Raycast Firefox extension installed and Firefox running?")
      - Generate id via crypto.randomUUID()
      - Create promise, set 2000ms timeout that deletes from map and rejects with Error("WebExtension response timeout (2s)")
      - Add to pendingRequests map: { resolve, reject, timer }
      - Call sendNativeMessageFn({ id, command, params })
      - Return promise
    - `handleNativeResponse(message)`:
      - Extract id from message
      - If message has `type === 'handshake'`: log handshake response info, return (not a request-response)
      - Look up id in pendingRequests
      - If not found: log warning "Response for unknown request ID", return
      - clearTimeout(timer), delete from map
      - If message.ok: resolve(message.data)
      - Else: reject(new Error(message.error || 'Unknown error from WebExtension'))
    - `sendVersionHandshake()`:
      - If !sendNativeMessageFn, return
      - Send { id: crypto.randomUUID(), type: 'handshake', version: require('../package.json').version, protocol: 1 }
      - Log "Version handshake sent"
    - Export: setSendNativeMessage, setNativeConnected, isNativeConnected, sendRequest, handleNativeResponse, sendVersionHandshake
  </action>

    - `node -e "const b = require('./native-host/src/bridge'); console.log(typeof b.sendRequest, typeof b.handleNativeResponse, typeof b.setSendNativeMessage)"` outputs "function function function"
    - Bridge timeout test: `node -e "const b = require('./native-host/src/bridge'); b.setSendNativeMessage(()=>{}); b.setNativeConnected(true); b.sendRequest('test').catch(e => console.log(e.message))"` outputs timeout error after 2s
    - Bridge response test: `node -e "const b = require('./native-host/src/bridge'); let sentMsg; b.setSendNativeMessage((m)=>{sentMsg=m}); b.setNativeConnected(true); b.sendRequest('test').then(d=>console.log(d.x)); setTimeout(()=>b.handleNativeResponse({id:sentMsg.id, ok:true, data:{x:42}}), 50)"` outputs "42"
  </verify>

  <name>Task 2: Create HTTP server with endpoints, wire host.js, and add install script</name>

  <action>
    Create `native-host/src/health.js`:
    - Import bridge (for isNativeConnected), package.json (for version)
    - `handleHealth(currentPort)`: returns the health object (does NOT write HTTP response)
    - Returns: { ok: true, data: { uptime: process.uptime(), firefoxConnected: bridge.isNativeConnected(), port: currentPort, version: pkg.version, pid: process.pid, memoryMB: Math.round(process.memoryUsage().rss / 1024 / 1024) }, meta: { timestamp: Date.now() } }
    - Export handleHealth

    Create `native-host/src/server.js`:
    - Import http, { URL }, logger, bridge (sendRequest), lifecycle (writePortFile), health (handleHealth)
    - Constants: BASE_PORT = 26394, MAX_PORT_ATTEMPTS = 10
    - Module state: server = null, currentPort = null, starting = false, startPromise = null
    - `ensureServer()`:
      - If server already running, return { server, port: currentPort }
      - If starting, return startPromise (prevents double-start)
      - Set starting = true, create and return startPromise
      - Call startServer(0), store result, set starting = false
    - `startServer(attempt)`:
      - If attempt >= MAX_PORT_ATTEMPTS: reject with error about port range exhausted
      - port = BASE_PORT + attempt
      - Create http.createServer(handleRequest)
      - On 'error': if EADDRINUSE, log warning, resolve(startServer(attempt + 1)). Otherwise reject.
      - server.listen(port, '127.0.0.1', callback)
      - On success: log "HTTP server listening" with port, call writePortFile(port), set currentPort, resolve
    - `handleRequest(req, res)`:
      - Parse URL, extract pathname and method
      - Set headers: Content-Type application/json, Access-Control-Allow-Origin *
      - Handle OPTIONS for CORS preflight: return 204 with Allow-Methods GET,POST,OPTIONS and Allow-Headers Content-Type
      - Route:
        - GET /health -> sendJSON(res, 200, handleHealth(currentPort))
        - GET /tabs -> handleGetTabs(req, res)
        - POST /switch -> handleSwitchTab(req, res)
        - else -> sendJSON(res, 404, { ok: false, error: 'Not found' })
      - Wrap in try/catch: on error, log it, sendJSON(res, 500, { ok: false, error: 'Internal server error. Check logs at ~/.raycast-firefox/logs/' })
    - `handleGetTabs(req, res)`:
      - try: const data = await sendRequest('list-tabs', {}) (no params needed for basic query)
      - sendJSON(res, 200, { ok: true, data, meta: { count: data.tabs?.length || 0, timestamp: Date.now() } })
      - catch: sendJSON(res, 502, { ok: false, error: err.message, meta: { timestamp: Date.now() } })
    - `handleSwitchTab(req, res)`:
      - Parse JSON body from req
      - Validate: tabId is required (return 400 if missing)
      - try: const data = await sendRequest('switch-tab', { tabId: body.tabId, windowId: body.windowId })
      - sendJSON(res, 200, { ok: true, data, meta: { timestamp: Date.now() } })
      - catch: sendJSON(res, 502, { ok: false, error: err.message, meta: { timestamp: Date.now() } })
    - `parseBody(req)`: promise that collects chunks and JSON.parse (from research examples)
    - `sendJSON(res, statusCode, body)`: res.writeHead(statusCode), res.end(JSON.stringify(body))
    - Export: ensureServer

    Update `native-host/host.js` to wire everything together:
    - Keep existing: logger require, console overrides, protocol require, stdin setup
    - Add requires: lifecycle, bridge, server
    - On startup (top-level async IIFE or main() function):
      1. lifecycle.killOldProcess() -- kill previous instance
      2. lifecycle.writePidFile() -- claim PID
      3. Register signal handlers: SIGTERM/SIGINT -> log shutdown, lifecycle.cleanupPidFile(), lifecycle.cleanupPortFile(), process.exit(0)
      4. Register process 'exit' handler -> lifecycle.cleanupPidFile(), lifecycle.cleanupPortFile()
    - Wire bridge:
      - bridge.setSendNativeMessage(sendNativeMessage) -- connect bridge to protocol output
      - Set nativeConnected = true initially (stdin starts connected when Firefox launches us)
      - bridge.setNativeConnected(true)
      - bridge.sendVersionHandshake() -- send handshake on initial connect
    - Wire stdin onMessage to bridge.handleNativeResponse (replace the simple logger from Plan 01)
    - Wire stdin 'end' handler: bridge.setNativeConnected(false), log "Firefox disconnected"
    - Lazy HTTP server (per user decision: "HTTP server starts lazily, not on host launch"):
      - Do NOT call ensureServer() during host initialization. The HTTP server must NOT start on host launch.
      - Instead, start the server when the first stdin data arrives from Firefox (the first native message). This is the signal that Firefox is connected and the host is ready to serve Raycast.
      - Implementation: In the stdin onMessage handler, before calling bridge.handleNativeResponse(), check a `serverStarted` flag. If false, call `server.ensureServer().catch(err => logger.error('Failed to start HTTP server', { error: err.message }))` and set `serverStarted = true`.
      - This ensures: (1) server does NOT start on host launch, (2) server starts as soon as Firefox is confirmed connected, (3) port file is only written when server is actually listening (clean signal to Raycast that the host is ready).
      - Log "Waiting for Firefox connection before starting HTTP server" at startup.
      - If ensureServer() fails across all 10 ports, log error but do not crash the host.

    - Print: "Host path: $RUN_SH"
  </action>
