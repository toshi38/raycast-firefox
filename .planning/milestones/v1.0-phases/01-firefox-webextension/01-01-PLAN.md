<task type="auto">
  <name>Task 2: Create background script with native messaging and all command handlers</name>
  <files>extension/background.js</files>
  <action>
Create `extension/background.js` implementing the complete background script. This is the core of the extension.

**Native Port Management:**
- `NATIVE_APP_NAME = "raycast_firefox"` (must match Phase 2 native host manifest name)
- Lazy `getPort()` function: creates port via `browser.runtime.connectNative()` only on first use (per user decision: lazy connection, not on Firefox startup)
- `port.onDisconnect` handler: log error, set port to null (per user decision: auto-reconnect by nulling and reconnecting on next request)
- `port.onMessage` listener: routes incoming messages to command handlers

**Message Protocol (per user decisions):**
- Inbound messages have shape: `{ id: string, command: string, params: object }`
- `id` field is echoed back in response for request-response correlation
- Success response: `{ id: string, ok: true, data: object }`
- Error response: `{ id: string, ok: false, error: string }`
- Unknown commands get error response (not silent failure)

**Command: list-tabs**
- Params: `{ page?: number, pageSize?: number }` (defaults: page=1, pageSize=500 per user decision)
- Calls `browser.tabs.query({})` to get ALL tabs (including private browsing per user decision)
- Resolves container metadata via `browser.contextualIdentities.query({})` -- wrap in try/catch because containers may not be enabled, degrade gracefully to null
- Maps each tab to: `{ id, windowId, title, url, favIconUrl, active, pinned, incognito, status, cookieStoreId, container }` where `container` is `{ name, color, icon }` or null
- Per user decision: return tabs immediately, don't wait for loading tabs to finish (`status` field captures loading state)
- Per user decision: `favIconUrl` is optional (use null if undefined)
- Apply pagination: slice array by page/pageSize
- Return: `{ tabs: [...], total: number, page: number, pageSize: number, hasMore: boolean }`

**Command: switch-tab**
- Params: `{ tabId: number }`
- Calls `browser.tabs.update(tabId, { active: true })` to activate tab
- Calls `browser.windows.update(tab.windowId, { focused: true })` to focus window
- Return: `{ tabId, windowId }`
- Error if tab not found (let browser API throw, catch in handler)

**Command: close-tab**
- Params: `{ tabId: number }`
- Calls `browser.tabs.remove(tabId)`
- Return: `{ tabId }`
- Error if tab not found (let browser API throw, catch in handler)

**Message routing:**
- Main `handleMessage(message)` async function with switch on `message.command`
- Wraps each handler in try/catch for consistent error responses
- Posts response back via `port.postMessage()`

**Debug support (for Plan 02 debug page):**
- Maintain a circular buffer of recent messages (last 50 inbound + outbound) as `recentMessages` array
- Each entry: `{ timestamp, direction: 'in'|'out', message }`
- Expose connection state and message log via `browser.runtime.onMessage` listener for internal extension page communication
- Handle messages like `{ type: "get-debug-info" }` from debug page, respond with `{ connected: boolean, recentMessages: [...], nativeAppName: string }`

**Code style:**
- Use `const` and arrow functions where appropriate
- All functions that call browser APIs should be async
- Log connection events to console for debugging: `console.log("[Raycast Firefox]", ...)`
