<objective>
Create the favicon caching module and HTTP endpoint on the native host so the Raycast extension can fetch cached favicons without hitting external services directly.

Purpose: The native host acts as a caching proxy for favicon data. It fetches favicon images from their HTTPS URLs (as provided by Firefox's favIconUrl), converts them to base64 data URIs, and caches them in memory and on disk. This avoids having the Raycast extension make external network calls and provides fast cached responses.

Output: New favicon-cache.js module with in-memory + disk cache, and a GET /favicon endpoint on server.js.
</objective>

**Module structure:**

Constants:
- `CACHE_DIR`: `path.join(os.homedir(), '.raycast-firefox', 'favicons')`
- `MAX_MEMORY_ENTRIES`: 500
- `MAX_AGE_MS`: 7 * 24 * 60 * 60 * 1000 (7 days)
- `FETCH_TIMEOUT_MS`: 5000 (5 second timeout for favicon fetches)

State:
- `memoryCache`: `new Map()` -- key: favicon URL string, value: `{ dataUri, timestamp }`

Functions:

1. `cacheKey(url)` -- SHA-256 hash of the URL string, returned as hex. Used for disk filenames.

2. `get(url)` -- Synchronous lookup. Check memory cache first (return `dataUri` if found and not expired). Then check disk cache (`CACHE_DIR/<hash>` file). If found on disk and not expired (check file mtime), promote to memory cache and return. Return `null` on miss. Use try/catch around fs operations.

3. `set(url, dataUri)` -- Write to both memory and disk. Memory: evict oldest entry if at MAX_MEMORY_ENTRIES (delete first key from Map iteration). Disk: `fs.mkdirSync(CACHE_DIR, { recursive: true })` then `fs.writeFileSync(path.join(CACHE_DIR, cacheKey(url)), dataUri)`.

4. `async fetchAndCache(url)` -- Fetch the favicon URL via Node.js global `fetch` (available in Node 18+). Set a timeout using AbortController. On success, convert response to base64 data URI:
   ```javascript
   const response = await fetch(url, { signal: AbortSignal.timeout(FETCH_TIMEOUT_MS) });
   if (!response.ok) return null;
   const contentType = response.headers.get('content-type') || 'image/x-icon';
   const buffer = Buffer.from(await response.arrayBuffer());
   const dataUri = `data:${contentType};base64,${buffer.toString('base64')}`;
   set(url, dataUri);
   return dataUri;
   ```
   Wrap everything in try/catch, return `null` on any error. Log errors at warn level.

5. `getOrFetch(url)` -- Convenience function: `get(url)` first, if null then `await fetchAndCache(url)`. Returns dataUri string or null.

Import logger: `const { logger } = require('./logger');`

Exports: `{ get, set, fetchAndCache, getOrFetch, CACHE_DIR }`

3. Create `handleGetFavicon` function:
   ```javascript
   async function handleGetFavicon(req, res, searchParams) {
     const url = searchParams.get('url');
     if (!url) {
       sendJSON(res, 400, { ok: false, error: 'url query parameter is required' });
       return;
     }

     try {
       const dataUri = await faviconCache.getOrFetch(url);
       if (dataUri) {
         sendJSON(res, 200, { ok: true, data: { dataUri } });
       } else {
         sendJSON(res, 404, { ok: false, error: 'Favicon not available' });
       }
     } catch (err) {
       logger.error({ err: err.message, url }, 'Favicon fetch error');
       sendJSON(res, 502, { ok: false, error: 'Failed to fetch favicon' });
     }
   }
   ```

The endpoint accepts `GET /favicon?url=<encoded-favicon-url>` and returns `{ ok: true, data: { dataUri: "data:image/...;base64,..." } }` or an error.
