Add favicon fetching to search-tabs.tsx. The approach: after loading tabs, batch-fetch favicons from the native host's `/favicon?url=<encoded>` endpoint for all tabs that have a `favIconUrl`. Store favicon data URIs in component state and use them as List.Item icons.

**Add favicon state and fetching:**

1. Add a `useState` for favicon data: `const [favicons, setFavicons] = useState<Record<string, string>>({})` -- keyed by favIconUrl, value is data URI string.

2. Add a `useEffect` that runs when `tabs` changes. For each tab with a non-empty `favIconUrl`, fetch from the native host:
   ```typescript
   useEffect(() => {
     if (!tabs || tabs.length === 0) return;

     // Collect unique favIconUrls that we haven't fetched yet
     const urlsToFetch = [...new Set(
       tabs
         .filter(t => t.favIconUrl && !favicons[t.favIconUrl])
         .map(t => t.favIconUrl as string)
     )];

     if (urlsToFetch.length === 0) return;

     // Fetch all favicons in parallel (fire-and-forget, update state as they arrive)
     const results: Record<string, string> = {};
     let settled = 0;

     urlsToFetch.forEach(async (url) => {
       try {
         const res = await fetch(`http://127.0.0.1:${port}/favicon?url=${encodeURIComponent(url)}`);
         if (res.ok) {
           const json = await res.json() as { ok: boolean; data?: { dataUri: string } };
           if (json.ok && json.data?.dataUri) {
             results[url] = json.data.dataUri;
           }
         }
       } catch {
         // Favicon fetch failed -- fallback icon will be used
       } finally {
         settled++;
         if (settled === urlsToFetch.length) {
           setFavicons(prev => ({ ...prev, ...results }));
         }
       }
     });
   }, [tabs]);
   ```

3. Add import for `useState, useEffect` from "react" at top of file.

**Update the icon in List.Item:**

Create a helper function `getTabIcon(tab: Tab): Image.ImageLike` that:
- Checks `favicons[tab.favIconUrl]` -- if present, return the data URI string
- Otherwise returns `getFallbackIcon(tab)` (the existing fallback from Plan 01)

Update the List.Item `icon` prop: `icon={getTabIcon(tab)}`

**Important considerations:**
- Favicons load asynchronously -- the list shows fallback icons immediately, then updates with real favicons as they load. This is good UX.
- Do NOT block rendering on favicon fetches. The list should appear instantly with fallback icons.
- Keep all existing functionality (sorting, accessories, keywords, actions) unchanged from Plan 01.
- The favicon fetch is fire-and-forget -- failures silently fall back to the letter avatar/Firefox icon.
