  <action>
**Debug Page (per user decision: hidden debug page for troubleshooting connection status and recent messages):**

Create `extension/debug.html`:
- Simple, functional HTML page (no frameworks)
- Title: "Raycast Firefox - Debug"
- Sections: Connection Status, Recent Messages
- Connection status display: green/red indicator with "Connected"/"Disconnected" text
- Message log: scrollable list showing recent messages with timestamp, direction (in/out), and message content (JSON formatted)
- Auto-refresh button and manual refresh button
- Include `debug.js` script tag
- Minimal inline CSS for readability (monospace font for message log, color-coded direction indicators)

Create `extension/debug.js`:
- On page load, call `browser.runtime.sendMessage({ type: "get-debug-info" })` to get current state from background script
- Render connection status (connected boolean -> green/red indicator)
- Render native app name
- Render recent messages list: for each message, show timestamp (formatted), direction arrow (-> for outbound, <- for inbound), and pretty-printed JSON
- Add "Refresh" button that re-fetches debug info
- Add auto-refresh toggle (poll every 2 seconds when enabled)
- Handle errors gracefully (if background script not responding, show "Unable to connect to background script")

**Update manifest.json:**
- Add debug.html as an extension page. No `web_accessible_resources` needed -- extension pages are accessible via `moz-extension://[uuid]/debug.html` by default in MV2. Just ensure the file exists in the extension directory.

**Create `extension/package.json`:**
```json
{
  "name": "raycast-firefox-extension",
  "version": "1.0.0",
  "private": true,
  "description": "Firefox WebExtension for Raycast tab management",
  "scripts": {
    "lint": "web-ext lint --source-dir .",
    "start": "web-ext run --source-dir . --firefox=firefox",
    "build": "web-ext build --source-dir . --overwrite-dest"
  },
  "devDependencies": {
    "web-ext": "^9.0.0"
  }
}
```

Run `cd extension && npm install` to install web-ext.

Run `cd extension && npm run lint` to validate the extension structure. Fix any lint errors.

Note: Add `extension/node_modules/` and `extension/web-ext-artifacts/` to a `.gitignore` at the project root (or extension root).
