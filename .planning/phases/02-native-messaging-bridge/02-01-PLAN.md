<task type="auto">
  <name>Task 2: Create logger module, protocol module, and host.js entry point</name>
  <files>
    native-host/src/logger.js
    native-host/src/protocol.js
    native-host/host.js
  </files>
  <action>
    Create `native-host/src/logger.js`:
    - Export a pino logger instance configured with pino-roll transport
    - Log directory: path.join(os.homedir(), '.raycast-firefox', 'logs')
    - pino-roll options: file = LOG_DIR/host.log, size = '5m', frequency = 'daily', mkdir = true, limit.count = 5
    - Logger level: 'info' (captures info, warn, error -- not debug/trace)
    - Custom formatter: level label instead of numeric
    - Export the logger AND a function to get the log directory path (for error messages to users)

    Create `native-host/src/protocol.js`:
    - Export `createDecoder(onMessage)` -- returns a function to call with each stdin data chunk
      - Accumulates data in a buffer
      - Reads 4-byte UInt32LE length prefix
      - Checks 512KB size limit -- if exceeded, log error via passed logger, reset buffer state, skip message
      - When full message body accumulated, JSON.parse and call onMessage(parsedObj)
      - Handles partial reads (message split across multiple data events)
      - Handles multiple messages in one chunk
    - Export `encode(message)` -- takes a JS object, returns a Buffer with 4-byte length prefix + JSON
      - Check 1MB limit on outbound (Firefox's limit)
      - Return Buffer containing: [4-byte UInt32LE length][UTF-8 JSON bytes]
    - Do NOT import logger directly -- accept it as parameter or use a setter, to avoid circular dependencies

    Create `native-host/host.js`:
    - Require logger from src/logger.js FIRST (before anything else)
    - Override console.log, console.error, console.warn to redirect to logger (prevents stdout/stderr corruption)
    - Require protocol from src/protocol.js
    - Log startup event: "Native messaging host starting" with version from package.json
    - Set up stdin reading:
      - Do NOT call process.stdin.setEncoding() (must stay in raw Buffer mode)
      - Create decoder via createDecoder(onMessage)
      - process.stdin.on('data', decoder)
      - process.stdin.on('end', () => { mark native connection as disconnected; log event; do NOT exit })
      - process.stdin.on('error', (err) => { log error; do NOT exit })
    - Export a `sendNativeMessage(msg)` function that uses protocol.encode() and writes to process.stdout
    - Export a `isNativeConnected()` function that returns the connection state
    - Export a `setSendNativeMessage()` or make sendNativeMessage available to bridge module
    - Set up process signal handlers: SIGTERM and SIGINT log shutdown event (but don't exit yet -- lifecycle module in Plan 02 handles PID cleanup)
    - For now, onMessage just logs received messages (bridge module in Plan 02 will handle routing)
    - Log "Host ready, waiting for native connection" after setup

    CRITICAL anti-patterns to avoid:
    - NEVER write to process.stdout except via protocol.encode() for native messages
    - NEVER call process.stdin.setEncoding()
    - NEVER use console.log/error/warn without the override in place
    - NEVER exit on stdin end/error
  </action>
  <verify>
    - `node -e "const l = require('./native-host/src/logger'); console.log(typeof l.logger.info)"` outputs "function"
    - `node -e "const p = require('./native-host/src/protocol'); const enc = p.encode({test:1}); console.log(enc.length > 4, enc.readUInt32LE(0) === enc.length - 4)"` outputs "true true"
    - `node -e "const p = require('./native-host/src/protocol'); let received; const dec = p.createDecoder((msg) => { received = msg; }); const buf = p.encode({hello:'world'}); dec(buf); console.log(received.hello)"` outputs "world"
    - `node -e "const p = require('./native-host/src/protocol'); let msgs=[]; const dec = p.createDecoder((m)=>msgs.push(m)); const b1=p.encode({a:1}); const b2=p.encode({b:2}); dec(Buffer.concat([b1,b2])); console.log(msgs.length)"` outputs "2" (multi-message in single chunk)
    - `node -e "const p = require('./native-host/src/protocol'); let msgs=[]; const dec = p.createDecoder((m)=>msgs.push(m)); const buf=p.encode({x:1}); dec(buf.slice(0,2)); dec(buf.slice(2)); console.log(msgs.length)"` outputs "1" (partial read handling)
  </verify>
