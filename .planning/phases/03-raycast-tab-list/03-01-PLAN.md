  <action>
Create `raycast-extension/src/search-tabs.tsx` as the single entry point for the "Search Firefox Tabs" command.

The file must implement:

1. **Tab type interface** matching the HTTP API response from Phase 2:
```typescript
interface Tab {
  id: number;
  windowId: number;
  title: string;
  url: string;
  favIconUrl: string | null;
  active: boolean;
  pinned: boolean;
  incognito: boolean;
  status: string;
  cookieStoreId: string;
  container: { name: string; color: string; icon: string } | null;
}

interface TabsResponse {
  ok: boolean;
  data: {
    tabs: Tab[];
    total: number;
    page: number;
    pageSize: number;
    hasMore: boolean;
  };
  meta: { count: number; timestamp: number };
}
```

2. **Port discovery function** `getPort()`:
   - Reads `~/.raycast-firefox/port` synchronously via `readFileSync`
   - Parses content as integer
   - Falls back to `26394` on any error (file missing, parse failure)
   - Called once at module level or via a lazy constant (NOT inside the component re-render cycle)

3. **URL keyword extraction** `urlKeywords(url: string): string[]`:
   - Parse URL with `new URL(url)`
   - Extract hostname, split hostname by `.` for individual parts
   - Extract pathname segments by splitting on `/` and filtering empty strings
   - Return `[hostname, ...hostParts, ...pathParts]`
   - Return `[url]` on parse failure (for non-standard URLs like `about:home`)
   - This is CRITICAL for TABS-01 -- without keywords, typing a URL in Raycast search will NOT match any tabs because Raycast's built-in filter does NOT search `subtitle`

4. **SearchTabs component** (default export):
   - Use `useFetch<TabsResponse>` from `@raycast/utils` with URL `http://127.0.0.1:${port}/tabs`
   - Set `keepPreviousData: true` to avoid flickering
   - Extract tabs array: `data?.data?.tabs ?? []`
   - Render `<List>` with `isLoading={isLoading}` and `searchBarPlaceholder="Search Firefox tabs..."`
   - Do NOT set `onSearchTextChange` -- let Raycast's built-in fuzzy filtering handle search (TABS-01)
   - Do NOT set `filtering={false}` -- the default is built-in filtering which is what we want
   - For each tab, render `<List.Item>` with:
     - `key={String(tab.id)}`
     - `title={tab.title || "Untitled"}`
     - `subtitle={tab.url}` (TABS-02: URL displayed as subtitle)
     - `keywords={urlKeywords(tab.url)}` (TABS-01: URL searchable via keywords)

Do NOT add any ActionPanel or actions yet -- those are Phase 4 (Tab Switching). Keep the component minimal and focused on display + search.
