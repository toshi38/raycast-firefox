  <action>
Create `.github/workflows/release.yml` with:

**Trigger:** `push` to `main` branch only.

**Concurrency:** `${{ github.workflow }}-${{ github.ref }}` to prevent duplicate release runs.

**Permissions:** `contents: write`, `pull-requests: write` (needed for creating releases and version PRs).

**Runner:** `macos-latest`

**Job: `release`**

Steps:
1. `actions/checkout@v4`
2. `actions/setup-node@v4` with `node-version: 20` and `cache: npm`
3. `npm ci`
4. Build native host: `npm run build --workspace=native-host`
5. **Changesets action:**
   ```yaml
   - name: Create Release Pull Request or Publish
     id: changesets
     uses: changesets/action@v1
     with:
       publish: npx changeset publish
       version: npx changeset version
       commit: "chore: version packages"
       title: "chore: version packages"
       createGithubReleases: false
     env:
       GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
   ```
   CRITICAL: `createGithubReleases: false` because the built-in release creation does NOT work for private packages. We handle it ourselves below.

6. **Generate SHA256 checksum** (conditional on `steps.changesets.outputs.published == 'true'`):
   ```bash
   cd native-host/dist
   shasum -a 256 host.bundle.js run.sh > SHA256SUMS.txt
   shasum -a 256 -c SHA256SUMS.txt
   ```
   Uses `shasum -a 256` (macOS built-in, not `sha256sum` which is Linux). The round-trip verify (`-c`) confirms the checksums match what was just generated.

7. **Get native-host version** (conditional on published):
   ```bash
   VERSION=$(node -p "require('./native-host/package.json').version")
   echo "version=$VERSION" >> "$GITHUB_OUTPUT"
   echo "tag=native-host@$VERSION" >> "$GITHUB_OUTPUT"
   ```

8. **Create GitHub Release** (conditional on published):
   ```yaml
   - name: Create GitHub Release
     if: steps.changesets.outputs.published == 'true'
     uses: softprops/action-gh-release@v2
     with:
       tag_name: ${{ steps.nh-version.outputs.tag }}
       name: native-host@${{ steps.nh-version.outputs.version }}
       generate_release_notes: false
       files: |
         native-host/dist/host.bundle.js
         native-host/dist/run.sh
         native-host/dist/SHA256SUMS.txt
   ```

**IMPORTANT ANTI-PATTERNS TO AVOID:**
- Do NOT use a PAT instead of GITHUB_TOKEN (causes infinite workflow triggers)
- Do NOT create a separate tag-triggered workflow (GITHUB_TOKEN pushes don't trigger workflows)
- Do NOT set `createGithubReleases: true` (silently produces no releases for private packages)
- Do NOT use `npm install` instead of `npm ci`

**OPEN QUESTION: `published` output for private packages.**
The research identified uncertainty about whether `changesets/action` sets `published: true` for private-package-only repos. Add a comment in the workflow noting this. If `published` never becomes `true`, the fallback approach is to detect new tags via `git tag --points-at HEAD` and conditionally create the release. For now, trust the documented behavior and plan to test once the workflow runs.
