  <action>
Create `raycast-extension/src/lib/installer.ts` that exports a single `installNativeHost` function and supporting internals.

**Public API:**

```typescript
export interface InstallResult {
  version: string;       // e.g. "1.1.1"
  installDir: string;    // ~/.raycast-firefox/bin/
  symlinkPath: string;   // ~/.raycast-firefox/node
}

export async function installNativeHost(
  onProgress: (message: string) => void
): Promise<InstallResult>;
```

The `onProgress` callback is called with step messages ("Downloading native host...", "Verifying checksum...", "Installing files...") so the caller can update toast.message.

**Internal implementation (per user decisions):**

1. **getLatestNativeHostRelease()** - Fetch `https://api.github.com/repos/toshi38/raycast-firefox/releases/latest` with:
   - Headers: `Accept: application/vnd.github.v3+json`, `User-Agent: raycast-firefox-setup`
   - `AbortSignal.timeout(10000)` for 10s timeout
   - Handle 403 → throw "GitHub rate limit reached. Please try again in a few minutes."
   - Handle non-ok → throw "Failed to check for updates (HTTP {status})"
   - Parse response: extract `tag_name` (format: "native-host@X.Y.Z"), build `Map<string, string>` of asset name → browser_download_url
   - Extract version from tag_name by stripping "native-host@" prefix
   - **Important:** The `/releases/latest` endpoint returns the single most recent release. If the latest release is `extension@1.0.2` (not a native-host release), the assets won't include host.bundle.js. To handle this, use the `/releases` endpoint and find the first release whose `tag_name` starts with "native-host@". Alternatively, validate that the required assets exist and throw a clear error if not. Choose whichever is simpler, but document WHY in a code comment.

2. **downloadAsset(url: string)** - Download a single asset via `fetch(url)` (redirects followed by default). Return `Buffer.from(await res.arrayBuffer())`. Throw on non-ok response.

3. **sha256(buffer: Buffer)** - `createHash("sha256").update(buffer).digest("hex")`

4. **parseSha256Sums(content: string)** - Parse the "hash  filename" format (double-space separator from `shasum -a 256`). Return `Map<string, string>` of filename → hash.

5. **verifyChecksum(buffer: Buffer, filename: string, checksums: Map<string, string>)** - Compare sha256(buffer) against checksums.get(filename). Throw with clear message on mismatch or missing entry.

6. **installNativeHost(onProgress) flow:**
   - `onProgress("Downloading native host...")`
   - Fetch latest release info
   - Validate required assets exist: host.bundle.js, run.sh, SHA256SUMS.txt
   - Download all three assets in parallel via `Promise.all`
   - `onProgress("Verifying checksum...")`
   - Parse SHA256SUMS.txt content (it was downloaded as Buffer, convert to string)
   - Verify host.bundle.js and run.sh against checksums (SHA256SUMS.txt itself is not checksummed)
   - `onProgress("Installing files...")`
   - **Atomic install per user decision:**
     - Create temp dir inside `environment.supportPath` (use `mkdtempSync`)
     - Write host.bundle.js and run.sh to temp dir
     - `chmodSync(run.sh, 0o755)` in temp dir
     - Create final install dir `~/.raycast-firefox/bin/` with `mkdirSync({ recursive: true })`
     - Move/copy files from temp to install dir (use `renameSync` if same filesystem, fall back to `copyFileSync` + `unlinkSync`)
     - Write `version.txt` LAST as install-complete marker (contains just the version string, e.g. "1.1.1")
     - Clean up temp dir with `rmSync({ recursive: true })`
   - **Create Node.js symlink per user decision:**
     - Path: `~/.raycast-firefox/node`
     - Target: `process.execPath` (Raycast's bundled Node.js in production, system Node in dev)
     - Remove existing symlink if present (check with `lstatSync` + `isSymbolicLink()`)
     - Create with `symlinkSync(process.execPath, symlinkPath)`
     - Always create regardless of dev/production mode
   - Return `{ version, installDir, symlinkPath }`

**Error handling:**
- All errors should have clear, user-facing messages
- Network errors: "Check your internet connection and try again"
- Rate limit: "GitHub rate limit reached. Please try again in a few minutes."
- Checksum mismatch: "Checksum mismatch for {filename}. The download may be corrupted. Please try again."
- File system errors: let them propagate naturally (caller handles)

**Imports:** Only use Node.js built-ins (`crypto`, `fs`, `os`, `path`) and `@raycast/api` (`environment`). No new dependencies.

Rewrite `raycast-extension/src/setup-bridge.tsx` to integrate the installer before the manifest-write step. The command stays as `no-view` mode (per user decision).

**New flow (per user decisions from CONTEXT.md):**

```
1. Pre-flight: Firefox installed?
2. Show animated toast "Installing Firefox Bridge"
3. Call installNativeHost(msg => toast.message = msg)
   - Toast updates: "Downloading native host..." → "Verifying checksum..." → "Installing files..."
4. Resolve native host path (will now find ~/.raycast-firefox/bin/run.sh)
5. Generate and write manifest
6. Validate manifest
7. Chain verification with post-install guidance
```

**Toast UX (per user decisions):**
- Initial: `Toast.Style.Animated`, title "Installing Firefox Bridge"
- Progress: update `toast.message` via the onProgress callback
- On install error: `Toast.Style.Failure` with specific error message + "Try Again" primary action that re-runs setup via `launchCommand({ name: "setup-bridge", type: LaunchType.UserInitiated })`
- On full chain success (reachable + tabsOk): `showHUD("Firefox integration ready!")`
- On partial success (host installed, Firefox not running): `Toast.Style.Success`, title "Firefox Bridge Installed", message "Open Firefox and install the companion extension", primary action "Install Firefox Extension" → `open(AMO_URL)`
- On partial success (host installed, host reachable but tabs fail — extension not connected): `Toast.Style.Success`, title "Firefox Bridge Installed", message "Install the Firefox companion extension to connect", primary action "Install Firefox Extension" → `open(AMO_URL)`
- On partial success (host installed, host not reachable, Firefox IS running): `Toast.Style.Success`, title "Firefox Bridge Installed", message "Restart Firefox to activate the native host", no AMO link needed (Firefox is running, extension may be installed)

**AMO URL constant:** `https://addons.mozilla.org/en-US/firefox/addon/raycast-tab-manager-for-firefox/`

**Error handling for installer step:**
- Wrap `installNativeHost()` in try/catch
- On error: update toast to Failure style with the error message
- Add "Try Again" primary action: `{ title: "Try Again", onAction: () => launchCommand({ name: "setup-bridge", type: LaunchType.UserInitiated }) }`
- Return early (don't proceed to manifest write)

**Post-install guidance (per user decisions):**
- Use the existing `isFirefoxRunning()` from `lib/errors.ts` to distinguish "Firefox not running" vs "extension not connected"
- After chain verification, determine state:
  - `chain.reachable && chain.tabsOk` → full success → `showHUD("Firefox integration ready!")`
  - `!chain.reachable && !isFirefoxRunning()` → Firefox not running → success toast with "Open Firefox..." message + AMO action
  - `!chain.reachable && isFirefoxRunning()` → host not started yet (needs Firefox restart) → success toast with "Restart Firefox..." message
  - `chain.reachable && !chain.tabsOk` → extension not connected → success toast with "Install the Firefox companion extension..." message + AMO action

**Imports to add:**
- `import { installNativeHost } from "./lib/installer";`
- `import { isFirefoxRunning } from "./lib/errors";`
- `import { LaunchType, launchCommand, open } from "@raycast/api";`

**Key: The manifest write and validation steps remain unchanged.** The installer populates `~/.raycast-firefox/bin/` so that `resolveNativeHostPath()` finds the production path. The rest of the existing flow continues as before.
