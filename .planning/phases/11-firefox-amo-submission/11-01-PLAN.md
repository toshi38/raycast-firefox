<task type="auto">
  <name>Task 1: Update manifest and upgrade web-ext</name>
  <files>extension/manifest.json, extension/package.json</files>
  <action>
1. Update `extension/manifest.json`:
   - Change `"name"` from `"Raycast Firefox"` to `"Raycast Tab Manager for Firefox"` (per user decision, AMO-03 compliance)
   - Add `"data_collection_permissions": { "required": ["none"] }` inside `browser_specific_settings.gecko` (per user decision, AMO-02 compliance). Place it after `"strict_min_version"`. Do NOT add an `"optional"` array.
   - Keep extension ID as `"raycast-firefox@lau.engineering"` (NEVER change this)
   - Keep all other fields unchanged

2. Update `extension/package.json`:
   - Change `"web-ext"` from `"^9.0.0"` to `"^9.4.0"`

3. Run `cd extension && npm install` to update the lockfile with the new web-ext version

4. Run `cd extension && npm run lint` to validate the updated manifest passes web-ext lint
   - NOTE: A warning about `data_collection_permissions` requiring Firefox 140+ is EXPECTED and acceptable (per Phase 1 decision). Only errors are blockers.

5. Run `cd extension && npm run build` to produce the zip

6. Inspect the built zip with `unzip -l extension/web-ext-artifacts/*.zip` to verify:
   - Contains: manifest.json, background.js, icons/, debug.html, debug.js
   - Does NOT contain: node_modules/, package.json, package-lock.json, .git/
   - If unwanted files are present, create `extension/web-ext-config.json` with `{"ignoreFiles": [...]}` to exclude them and rebuild

The CI workflow already has `npm run lint --workspace=extension` so no CI lint changes needed -- the upgraded web-ext will be used automatically on next PR.

7. Add extension zip build to `.github/workflows/release.yml`:
   - After the existing "Build native host bundle" step (in the `release` job), add a step to build the extension zip: `npm run build --workspace=extension`
   - Add the built zip to the `files:` list in the "Create GitHub Release" step so it's attached as a release asset alongside `host.bundle.js`, `run.sh`, and `SHA256SUMS.txt`
   - The zip path will be `extension/web-ext-artifacts/raycast_tab_manager_for_firefox-*.zip` — use a glob pattern in the `files:` list
   - This lets you download the zip from any GitHub Release and manually upload to AMO (stepping stone to DIST-02 full automation)
