Add a `handleCloseTab` function and route it in the native host HTTP server. This mirrors the existing `handleSwitchTab` pattern exactly, minus the osascript Firefox activation.

**New function (add after handleSwitchTab):**
```javascript
async function handleCloseTab(req, res) {
  let body;
  try {
    body = await parseBody(req);
  } catch (err) {
    sendJSON(res, 400, { ok: false, error: 'Invalid JSON body' });
    return;
  }

  if (!body.tabId && body.tabId !== 0) {
    sendJSON(res, 400, { ok: false, error: 'tabId is required' });
    return;
  }

  try {
    const data = await sendRequest('close-tab', { tabId: body.tabId });
    sendJSON(res, 200, { ok: true, data, meta: { timestamp: Date.now() } });
  } catch (err) {
    sendJSON(res, 502, { ok: false, error: err.message, meta: { timestamp: Date.now() } });
  }
}
```

**CRITICAL difference from handleSwitchTab:** Do NOT include the `execFile('osascript', ...)` block. The user wants Firefox to stay in the background.

**Route addition:** In the `handleRequest` function, add an `else if` clause for `/close`:
```javascript
} else if (pathname === '/close' && method === 'POST') {
  await handleCloseTab(req, res);
}
```
Add this AFTER the `/switch` route and BEFORE the 404 fallback.

Note: CORS preflight already covers POST in the existing OPTIONS handler -- no changes needed there.

**Import changes:**

**usePromise change:**
- Destructure `mutate` from the existing `usePromise(fetchAllTabs)` call:
  ```typescript
  const { data: tabs = [], isLoading, mutate } = usePromise(fetchAllTabs);
  ```

**New function (add after the existing `switchTab` function):**
```typescript
async function closeTab(tabId: number, mutate: MutatePromise<Tab[] | undefined>) {
  const toast = await showToast({ style: Toast.Style.Animated, title: "Closing tab..." });
  try {
    await mutate(
      fetch(`http://127.0.0.1:${port}/close`, {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ tabId }),
      }),
      {
        optimisticUpdate(data) {
          return data?.filter((t) => t.id !== tabId);
        },
      },
    );
    toast.style = Toast.Style.Success;
    toast.title = "Tab closed";
  } catch {
    toast.style = Toast.Style.Failure;
    toast.title = "Failed to close tab";
    toast.message = "Could not connect to Firefox";
  }
}
```

**Import the MutatePromise type:**
- Add `MutatePromise` to the `@raycast/utils` import:
  ```typescript
  import { getAvatarIcon, usePromise, MutatePromise } from "@raycast/utils";
  ```

**ActionPanel modification:**
Replace the existing single-Action ActionPanel with sectioned ActionPanel:
```tsx
<ActionPanel>
  <ActionPanel.Section>
    <Action
      title="Switch to Tab"
      icon={Icon.Globe}
      onAction={() => switchTab(tab.id, tab.windowId)}
    />
  </ActionPanel.Section>
  <ActionPanel.Section>
    <Action
      title="Close Tab"
      icon={Icon.Trash}
      style={Action.Style.Destructive}
      shortcut={Keyboard.Shortcut.Common.Remove}
      onAction={() => closeTab(tab.id, mutate)}
    />
  </ActionPanel.Section>
</ActionPanel>
```

**Key design decisions (per research):**
- `Action.Style.Destructive` gives red text styling for the close action
- `Keyboard.Shortcut.Common.Remove` maps to Ctrl+X (standard Raycast shortcut for remove)
- No confirmation dialog (matches Firefox behavior -- closing a tab is not data-destructive)
- `mutate` with `optimisticUpdate` removes the tab instantly from the local array; `shouldRevalidateAfter` defaults to `true` so the full list re-fetches after the mutation completes (updates active tab state, etc.)
- Do NOT call `closeMainWindow` -- the user should stay in Raycast to close more tabs
