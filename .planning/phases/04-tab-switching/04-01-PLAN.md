**New function to add (before the component):**
```typescript
async function switchTab(tabId: number, windowId: number) {
  await closeMainWindow({ clearRootSearch: true });
  try {
    const response = await fetch(`http://127.0.0.1:${port}/switch`, {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ tabId, windowId }),
    });
    if (!response.ok) {
      const body = await response.json().catch(() => ({ error: "Unknown error" }));
      await showHUD(`Switch failed: ${body.error}`);
    }
  } catch {
    await showHUD("Could not connect to Firefox");
  }
}
```

CRITICAL: `closeMainWindow` MUST be called BEFORE the fetch POST, not after. This is the "close-first" pattern -- the user perceives instant switching. Do NOT await the HTTP response before closing.

**Modifications to List.Item:**
- Add `icon={getFavicon(tab.url)}` prop for website favicons
- Add `actions` prop with an `ActionPanel` containing a single primary `Action`:
  - title: "Switch to Tab"
  - icon: `Icon.Globe`
  - onAction: `() => switchTab(tab.id, tab.windowId)`

The "Switch to Tab" Action MUST be the FIRST child of ActionPanel so it becomes the primary action (triggered by Enter key).

IMPORTANT: Always include both `tabId` AND `windowId` in the POST body. The windowId is available from the tab data and is required for multi-window switching.

Do NOT add getFavicon to Phase 5 scope -- it is being added here since it is one line and improves UX immediately. Phase 5 can later refine favicons (e.g., using Firefox's favIconUrl as primary source).
